# 갤러리 페이크 번역 프로젝트: V3 하이브리드 파이프라인 세션 리포트

**날짜:** 2026-03-03  
**상태:** 제미니 리소스 소진으로 인한 일시 중단 (복구 대기 중)

---

## 1. 현재 프로젝트 아키텍처 (V3: 하이브리드)
사용자님의 분석('나노 바나나 2')을 바탕으로, 기존의 모든 실패 설계를 폐기하고 가장 진보된 하이브리드 방식을 구축했습니다.

- **눈(Detector)**: `PaddleOCR` (오프라인, 무료, 일본어 세로/가로쓰기 텍스트 스파팅 1등 성능)
- **뇌(Translator)**: `Gemini 2.5 Flash` (조각 이미지를 직접 읽는 VLM 방식 + 안티-디텍션 난수 지연 로직)
- **손(Editor)**: `OpenCV`(Inpainting) + `Pillow`(Typesetting)

## 2. 주요 달성 사항 및 파일 목록
각 단계의 모듈은 상호 간섭을 줄이기 위해 독립된 파일로 완성되어 있습니다.

- `step1_detector.py`: `PaddleOCR`을 이용해 텍스트 박스만 기가 막히게 따내는 모듈 (완료)
- `step2_vlm_translator.py`: 따낸 박스 이미지를 각각 크롭하여 제미니에게 번역을 맡기는 모듈. **난수 지연(Jittering)**을 넣어 봇 탐지를 회피합니다. (완료)
- `step3_inpainting.py`: 원본 일본어를 무손실에 가깝게 지우는 식질 모듈 (완료)
- `step4_typesetting.py`: 말풍선 크기에 맞춰 한글을 자동 줄바꿈하고 테두리를 입히는 조판 모듈 (완료)
- `main.py`: 위 4단계를 엮어 폴더 내 모든 이미지를 자동 변환하는 통합 컨트롤러 (완료)

## 3. 남아있는 작업 (리소스 복구 시 즉시 실행)
현재 `main.py` 실행 시 모듈 이름 정의 오차로 `ImportError`가 발생한 상태에서 중단되었습니다. 작업 재개 시 다음 한 가지만 수행하면 됩니다.

1. **`main.py` 내 함수명 매칭**: `step3_inpainting.py`와 `step4_typesetting.py` 내의 함수명(`inpaint_image`, `typeset_image`)이 `main.py`에서 호출하는 이름과 완벽히 일치하는지 최종 점검 후 실행.
2. **실전 런칭**: `python main.py test_images` 실행하여 퀄리티 최종 확인.

## 4. 나노 바나나 2 벤치마킹 분석 결과
`analysis_results.md`에 상세 기록되어 있습니다. 핵심은 **"모델의 분업"**입니다. 제미니에게 좌표까지 시키면 60점이지만, 좌표는 전용 알고리즘이 따고 제미니는 읽기만 하면 90점이 나옵니다.
